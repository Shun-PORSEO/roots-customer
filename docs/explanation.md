## このアプリは何をするものか

**結婚式の準備タスクを、新郎新婦がLINEアプリから管理できるサービス**です。

- 新郎新婦 → LINEのトーク画面から起動して、自分のタスク一覧を確認・完了チェック
- プランナー → Googleスプレッドシートを開いて、全顧客の進捗を管理・タスクを追加・修正

---

## 全体の仕組みを「店舗」に例えると

```
新郎新婦（お客さん）
   ↓ LINEアプリから注文（タスク確認・完了ボタン）
──────────────────────────────────────────────
   フロントエンド（店頭のタブレット）
   → LIFFというLINEの仕組みで動くWebページ
   → 作る技術：Next.js（画面を作るツール）
──────────────────────────────────────────────
   ↑↓ 注文票を裏に渡す（API通信）
──────────────────────────────────────────────
   バックエンド（厨房）
   → GAS（Google Apps Script）というGoogleのプログラム機能
   → スプレッドシートを「データベース（データの保管庫）」として読み書きする
──────────────────────────────────────────────
   ↑↓ データを保管・更新
──────────────────────────────────────────────
   Googleスプレッドシート（冷蔵庫）
   → タスク一覧・顧客情報・進捗がすべてここに入っている
   → プランナーはここを直接開いて管理する
```

## なぜこの技術を選んだのか

### LINE LIFF（ラィフ）— 「新郎新婦がアプリをダウンロードしなくていい」から

LIFFとは「LINEの中でWebページを開く仕組み」です。
新郎新婦は普段使いのLINEを開くだけで利用できます。専用アプリのインストール不要。
さらにLINEのログイン情報を自動で使えるので、**IDやパスワードの入力が一切不要**です。

### Next.js — 「画面が速く、作りやすい」から

Next.jsはWebページを作るためのフレームワーク（部品セット）です。
スマートフォンで見やすい画面を効率よく作れます。

### Google Apps Script（GAS）— 「サーバーを契約しなくていい」から

GASはGoogleが無料で提供しているプログラム実行環境です。
普通なら「サーバー（コンピューター）を借りる」作業と費用が必要ですが、
GASを使えば**Googleのインフラをそのまま使える**ので、コストゼロで始められます。

### Googleスプレッドシート — 「プランナーが今すぐ使える」から

専用のデータベースソフトを導入すると、プランナーの操作習得に時間がかかります。
スプレッドシートなら多くのプランナーがすでに慣れており、**追加トレーニング不要**です。

## データの流れ（タスク完了ボタンを押したとき）

```
[新郎新婦がLINEでチェックをタップ]
         ↓
[フロントエンド（LIFF画面）]
  「タスクID + LINE ID + 完了フラグ」をGASに送信
         ↓
[GAS（バックエンド）]
  LINE IDが正しいか確認 → 問題なければスプレッドシートを更新
         ↓
[スプレッドシート]
  該当タスクの列を「完了」に書き換え
         ↓
[GASが「成功」と返信]
         ↓
[フロントエンド（LIFF画面）]
  チェックマークが付いた状態で画面が更新される
```

---

## セキュリティについて（安全性）

- LINE IDは「このユーザーを証明するカード」として機能します
- 他のユーザーのデータを取れないよう、GAS側でLINE IDを毎回チェックします
- LINEのログイン機構自体がセキュアなので、パスワード漏洩リスクがありません

---

## このアプリが完成したときの操作イメージ

1. 新郎新婦がLINEのトークルームにある「準備リスト」ボタンをタップ
2. 「shunさんの準備ダッシュボード」が表示される（初回のみニックネーム入力）
3. タスク一覧が「未完了」「完了」に分かれて表示される
4. タスクをタップすると、説明とマニュアルへのリンクが表示される
5. チェックボックスをタップすると即座に完了フラグが保存される

プランナーはスプレッドシートを開くだけで誰がどこで止まっているか確認できます。
