# Roots AI - 結婚式準備タスク管理システム 詳細仕様書

## 1. システム概要
本システムは、結婚式の準備をされるお客様（新郎新婦様）が、専用のシステム上で自身のやるべきこと（タスク）を直感的に管理しつつ、いつでも公式LINE上でAI専属プランナーに質問ができる「ハイブリッド型タスク管理＆AIアシスタントシステム」です。

お客様のLINEを通じた日常的なコミュニケーションの中にタスク管理を組み込むことで、準備における負担を軽減し、式場側とのコミュニケーションを円滑にすることを目的としています。

---

## 2. システム構成（アーキテクチャ）
本システムは、最先端のAI技術とサーバーレス環境を組み合わせた以下の技術スタックで構成されています。

- **フロントエンド（お客様画面）**: `Next.js` / `React` / `LINE LIFF`
- **バックエンド（API・Webhook制御）**: `Google Apps Script (GAS)`
- **データベース**: `Google Spreadsheet`
- **AIエンジン**: `Google Gemini 3.0 Flash Preview`
- **コミュニケーション基盤**: `LINE Messaging API`

### 2.1 アーキテクチャの特長
- **セキュリティの担保**: お客様の個人情報（氏名や連絡先）はスプレッドシートに保存せず、「LINEの内部ID」と「ニックネーム」のみでデータを管理する設計を採用し、セキュアな運用を実現しています。
- **データの一元管理**: クライアント（フロントエンド）とAI（LINEボット）の両方が、同一のバックエンド（GAS）を介してスプレッドシートを読み書きするため、常に「リアルタイムなデータ」がAIの頭脳に共有されます。
- **柔軟なAI連携**: Google Apps Script と Gemini API を直接サーバー内で連携させることで、高速かつ柔軟なAI応答を可能にしました。

## 3. 主要機能一覧

### 3.1 タスク管理機能（フロントエンド）
LINEアプリ内で立ち上がる専用画面（LIFF）にて、以下の機能を提供します。
1. **ユーザー自動認証**
   - LINEアカウントを用いた自動ログイン（パスワード等の入力不要）
   - 初回起動時のみ、システム上で利用する「ニックネーム」の登録
2. **タスク一覧と進捗管理**
   - 式場側が定義した「マスタータスク」の一覧表示
   - それぞれのタスクに対する「完了 / 未完了」のチェックボックス操作
   - 完了チェック時の、バックエンドへの非同期データ同期（楽観的UIによるスムーズな操作感）

### 3.2 AIプランナー機能（チャットボット）
LINEのトーク画面でメッセージを送信した際、AIが自動で回答する機能です。
1. **コンテキスト（文脈）対応アシスタント**
   - ユーザーからメッセージを受信した際、システムが裏側でそのユーザーの「現在のタスク完了状況」をデータベース（スプレッドシート）から瞬時に取得します。
   - 取得した進捗データをAI（Gemini）に渡し、「このお客様は現在〇〇のタスクが終わっていない」という状況を理解させた上で寄り添った回答を生成します。
2. **マニュアル参照応答**
   - 招待状の送付期限、席次表の確定ルール、BGMの著作権対応など、式場ごとの『基本マニュアル』をAIのプロンプトに内包しています。
   - お客様が「招待状はいつ発送ですか？」と尋ねれば、人に代わってAIが即座に正確なルールを返答します。
3. **トーン＆マナーの統一**
   - 回答は常に「明るく、丁寧で、絵文字を交えた親身なトーン」で出力されるよう、プロンプトレベルで調整されています。

## 4. データベース定義（Google Spreadsheet）
データはGoogle Spreadsheet上の3つのシートで一元管理されます。

1. **`customers` シート（顧客管理）**
   - `line_id`: LINEの内部識別ID
   - `nickname`: お客様のニックネーム
   - `created_at`: 初回登録日時
2. **`task_master` シート（タスク定義）**
   - `task_id`: タスクの一意なID
   - `title`: タスク名（例：招待状のリストアップ）
   - `description`: 補足説明
   - `due_offset_days`: 期日計算用のオフセット日数
   - `is_active`: タスクの有効/無効フラグ
3. **`task_progress` シート（進捗管理）**
   - `line_id`: 顧客ID
   - `task_id`: 紐づくタスクID
   - `is_done`: 完了フラグ（TRUE/FALSE）
   - `updated_at`: 最終操作日時

## 5. 今後の拡張性について（Phase 3 展望）
本システムのアーキテクチャは、拡張性を念頭に置いて設計されています。
将来的には以下の機能拡張がシームレスに行える構造となっています。

- **管理者用ダッシュボードの実装**: プランナー側が各ユーザーの進捗をPCブラウザから一覧確認できるNext.js製管理画面の追加。
- **式場別マニュアルの動的切り替え**: スプレッドシート上に「複数のマニュアル」を定義し、ユーザーごとにAIが読み込むベースマニュアルを切り替える機能の実装。
- **リマインド通知の自動化**: GASの時間主導型トリガーを用いた、期限の迫ったタスクの自動プッシュ通知（LINE メッセージの自動送信）。

## 6. プロトタイプ開発コスト概算（お見積もり参考）
本環境（Next.js × GAS × Spreadsheet × Gemini API）をゼロから構築し、本番投入可能なMVP（Minimum Viable Product / 最小構成のプロトタイプ）として完成させる場合の一般的な開発コスト試算です。
サーバーレスアーキテクチャを活用することで、初期費用およびランニングコストを大幅に抑える設計となっています。

| フェーズ・項目 | 作業内容の概要 | 概算費用（JPY） |
|---|---|---|
| **要件定義・基本設計** | 仕様策定、スプレッドシートDB設計、アーキテクチャ・UI設計 | 100,000 〜 150,000 |
| **フロントエンド開発** | Next.js/Reactによる画面構築、LINE LIFF認証連携、UX実装 | 200,000 〜 350,000 |
| **バックエンド・DB構築** | GAS/Spreadsheet環境構築、DB操作ロジック、リアルタイム同期 | 150,000 〜 250,000 |
| **AIモジュール連携** | LINE Webhook連携、Gemini APIへのコンテキスト注入、プロンプト調整 | 150,000 〜 250,000 |
| **テスト・QA** | セキュリティ検証、E2Eテスト、LINEボット動作検証 | 50,000 〜 100,000 |
| **合計（概算）** | プロトタイプ開発総額 | **¥650,000 〜 ¥1,100,000** |

※上記は開発初期費用（イニシャルコスト）の相場であり、月額保守費用やGoogle/LINE等のAPI利用料（従量課金）は含まれておりません。
※本構築手法では自社サーバーを立てないため、ランニングコストは実質的にAPIの通信料のみとなり、保守運用コストを極小化できるのが最大のメリットです。
